# Flujo de Autenticación en Frontend

## AuthProvider

`modules/auth/context/AuthProvider.tsx` es el proveedor de contexto que gestiona el estado de autenticación.

### Ciclo de vida

```
App inicia
    │
    ├── ¿Hay token en localStorage?
    │   │
    │   ├── Sí → GET /auth/profile → ¿válido?
    │   │       │                       │
    │   │       │   ├── Sí → carga roles + permisos → estado: autenticado
    │   │       │   └── No → limpia token → estado: no autenticado
    │   │
    │   └── No → estado: no autenticado
    │
    └── Usuario navega a ruta protegida
            │
            └── RequireAuth
                ├── loading → "Cargando sesión..."
                ├── !authenticated → redirect /login
                └── !hasRole → redirect /sin-acceso
```

### AuthContextValue

```typescript
{
  user: AuthUser | null           // { id, email }
  appUser: AppUser | null         // { id, schoolId, fullName, email, ... }
  token: string | null            // JWT
  roles: Role[]                   // Roles del usuario
  permissions: Permission[]       // Permisos del usuario
  loading: boolean                // Estado de carga inicial
  authError: string | null        // Error de autenticación
  isAuthenticated: boolean        // ¿Hay sesión activa?
  schoolId: string | null         // ID de la escuela del usuario
  login: (credentials) => Promise<void>
  register: (credentials) => Promise<void>
  logout: () => Promise<void>
  refreshAuth: () => Promise<void>
  hasRole: (roleKeys: UserRole[]) => boolean
  hasPermission: (permissionKey: string) => boolean
}
```

### apiClient

El cliente HTTP en `services/apiClient.ts`:
- Usa `fetch` nativo (sin axios)
- Envía `credentials: include`; el JWT propio permanece en una cookie `HttpOnly`
- Desenvuelve `response.data` automáticamente
- Lanza `ApiError` con status y message en errores

### Pantallas de autenticación

- **LoginPage:** Formulario de inicio de sesión (email + contraseña), con panel izquierdo de marca y enlace a registro
- **RegisterPage:** Formulario de registro (nombre escuela, slug, nombre completo, email, contraseña)
- **UnauthorizedPage:** Página "Acceso no autorizado" con opciones de volver al inicio o cerrar sesión
