# Módulo Attendance

**Responsabilidad:** Registro de asistencia diaria y por clase con lógica upsert.

| Método | Ruta | Roles | Descripción |
|--------|------|-------|-------------|
| GET | `/attendance` | Autenticado | Asistencia por clase (filtro sectionSubjectId + fecha) |
| GET | `/attendance/daily` | Autenticado | Asistencia diaria (filtro enrollmentId o sectionId+fecha) |
| GET | `/attendance/students` | Autenticado | Estudiantes para grid de asistencia (con estado actual) |
| GET | `/attendance/current-period` | Autenticado | Periodo académico activo |
| POST | `/attendance/upsert` | admin, director, coordinator, teacher | Crear o actualizar asistencia |

### Lógica upsert

El endpoint `POST /attendance/upsert` usa `INSERT ... ON CONFLICT UPDATE` (upsert) para crear o modificar registros sin duplicados. Soporta dos tipos:
- `class`: registra en `AttendanceClass` (asistencia por materia)
- `daily`: registra en `AttendanceDaily` (asistencia general del día)
