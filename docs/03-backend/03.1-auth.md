# Módulo Auth

**Responsabilidad:** Registro de escuela + admin, inicio de sesión (JWT), recuperación de contraseña mediante Supabase Auth y perfil propio.

**Archivos:**
- `auth.module.ts` — Importa PassportModule + JwtModule (8h expiración)
- `auth.controller.ts` — Endpoints REST
- `auth.service.ts` — Lógica de autenticación
- `strategies/jwt.strategy.ts` — Estrategia Passport JWT
- `strategies/jwt-auth.guard.ts` — Guard que usa la estrategia
- `dto/register.dto.ts`, `dto/login.dto.ts`

## Endpoints

| Método | Ruta | Auth | Roles | Descripción |
|--------|------|------|-------|-------------|
| POST | `/auth/register` | No | — | Crea escuela + admin, devuelve JWT |
| POST | `/auth/login` | No | — | Valida credenciales, devuelve JWT + roles + permisos |
| POST | `/auth/forgot-password` | No | — | Solicita a Supabase el correo de recuperación y responde un mensaje genérico |
| GET | `/auth/profile` | JWT | — | Devuelve perfil del usuario autenticado |

## Flujo JWT

1. **Login:** valida credenciales con Supabase Auth → carga roles y permisos de la escuela → firma JWT con `{ sub: userId, email, tokenVersion }` → lo guarda en la cookie `aulabase_session` (`HttpOnly`, `SameSite=Lax`, `Secure` en producción).
2. **Peticiones subsecuentes:** el navegador envía la cookie automáticamente; el JWT no está disponible para JavaScript.
3. **JwtStrategy.validate():** busca AppUser por `payload.sub`, verifica status ACTIVE, carga roles, devuelve `AuthenticatedUser { id, email, schoolId, roles[] }`
4. **RolesGuard:** opcionalmente verifica que `request.user.roles` incluya al menos uno de los roles requeridos
